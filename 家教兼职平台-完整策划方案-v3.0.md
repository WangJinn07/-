## 一、项目概述

### 1.1 产品定位

**家教需求信息聚合与实时沟通平台**（类BOSS直聘 + 内置聊天模式）

一个连接家长家教需求与杭州大学生的信息展示与沟通平台。管理员负责录入需求，大学生通过网页浏览需求，并可以**直接在平台内与管理员实时聊天**，表达接单意愿。

### 1.2 核心价值

- **对大学生**：快速浏览本地家教兼职 + 在线直接沟通管理员，不需要加微信
- **对家长**：需求由管理员整理发布，匹配更精准
- **对管理员**：智能文字解析快速录入 + 在线接收大学生咨询，高效协调对接

### 1.3 用户角色

| 角色 | 职责 | 使用方式 |
|------|------|----------|
| **家长** | 向管理员提交家教需求 | 微信/电话/其他方式 |
| **管理员**（运营人员） | 录入需求、在线回复大学生咨询、协调对接 | 管理后台（Web） |
| **大学生** | 浏览需求、在线联系管理员接单 | 网页浏览 + 内置聊天 |

### 1.4 核心亮点

✅ **智能文字解析录入** - 管理员直接粘贴文字，系统自动识别字段，支持批量录入  
✅ **内置聊天系统** - 大学生无需加微信，直接网页内沟通  
✅ **双编号管理** - 保留原始编号（KKJ231）+ 系统编号（D001）  
✅ **多维度筛选** - 按科目、区域、价格、时间快速定位  
✅ **实时消息通知** - 管理员实时收到大学生咨询

## 二、v3.0核心更新

### 2.1 相比v2.0的主要变化

#### 变化1：录入方式优先级调整

**v2.0方案**：
- 方式一：手动填写表单
- 方式二：OCR智能识别（图片）

**v3.0方案**（优化后）：
- 🥇 **优先级1：智能文字解析**（粘贴文字，自动识别字段，支持批量）
- 🥈 **优先级2：手动表单填写**（备用方案）
- 🥉 **优先级3：图片OCR**（后期扩展，暂不实现）

**优化理由**：
- 实际场景中，管理员从微信/QQ收到的需求多为**文字消息**，而非图片
- 文字解析比图片OCR更快、更准确、成本更低（无需调用第三方API）
- 支持一次粘贴多条需求，批量处理效率更高

#### 变化2：编号系统调整

**v2.0方案**：
- 系统自动生成唯一编号（D001、D002...）

**v3.0方案**（优化后）：
- **原始编号**（如KKJ231、KKJ229-2）：保留展示，方便管理员和家长对照原始需求
- **系统编号**（D001、D002...）：系统自动生成，用于内部管理、聊天引用、筛选排序

**优化理由**：
- 家长/管理员可能已经有自己的编号体系，直接保留避免混淆
- 系统编号用于技术实现，原始编号用于业务对照

#### 变化3：智能解析能力

**核心特点**：
- ✅ 支持**不规范格式**的文字识别（字段顺序可变、格式不统一）
- ✅ 使用**正则表达式 + 关键词匹配**，而非固定模板
- ✅ 识别后生成**预览结果**，管理员可人工修正后再发布
- ✅ 支持**批量识别**（一次粘贴多条需求）

## 三、业务流程

### 3.1 核心流程图（v3.0版 - 含智能解析）

```
家长提交需求给管理员
（微信/电话/QQ等渠道，文字描述）
    ↓
管理员复制需求文字
    ↓
【新功能】粘贴到系统，智能解析
（自动识别：年级、科目、地址、薪资、要求等）
    ↓
系统生成预览结果
（管理员可修正字段）
    ↓
确认发布，系统自动分配编号
（保留原始编号 + 生成系统编号）
    ↓
需求发布到平台
    ↓
大学生浏览需求列表
（无需注册，按科目/区域/价格筛选）
    ↓
点击查看需求详情
    ↓
【聊天功能】点击"在线咨询"按钮
    ↓
进入聊天界面，发送消息给管理员
（例如："我想接编号D001的单子"）
    ↓
管理员在后台实时收到消息
    ↓
管理员在线回复并协调对接
    ↓
后续由家长和大学生自行联系
```

### 3.2 详细流程说明

#### 阶段一：需求提交（家长 → 管理员）

家长通过微信/QQ/电话等方式把家教需求发给管理员，**通常是文字描述**。

**需求文字示例**：
```
编号：KKJ231＃暑假预约
信息：准六年级男生，语数外（1~5年级的内容过一遍），每天下午两小时或者隔一天上一次一次三小时，140/2小时，210/3小时
地址：上城区金色黎明
要求：女生，认真负责
```

#### 阶段二：智能解析录入（管理员操作）

**步骤1：粘贴文字**
- 管理员登录后台系统
- 进入"快速录入"页面
- 复制家长发来的需求文字
- 粘贴到文本框

**步骤2：智能识别**
- 点击"智能识别"按钮
- 系统自动解析字段：
  - 原始编号：KKJ231
  - 标签：暑假预约
  - 年级：准六年级 → 小学
  - 性别：男生
  - 科目：语数外 → [语文, 数学, 英语]
  - 薪资：140元/2小时, 210元/3小时
  - 时间：每天下午 或 隔一天
  - 地址：上城区金色黎明
  - 要求：女生，认真负责

**步骤3：预览与修正**
- 系统生成预览结果（结构化表格）
- 管理员检查识别是否准确
- 如有错误，手动修正字段
- 可以删除不需要的需求

**步骤4：批量发布**
- 确认无误后，点击"批量发布"
- 系统自动生成系统编号（D001、D002...）
- 需求发布到平台

#### 阶段三：需求浏览（大学生）

**无需注册，直接访问网页**

**首页展示**：
- 所有需求列表（卡片式）
- 每个卡片显示：
  - 系统编号（D001）+ 原始编号（KKJ231）
  - 标题（如"小学语数外家教"）
  - 科目、年级
  - 薪资（如"140元/2小时"）
  - 区域（如"上城区"）
  - 发布时间

**筛选功能**：
- 📍 按区域筛选
- 💰 按薪资范围筛选
- 📚 按科目筛选
- 🎓 按年级筛选

#### 阶段四：查看详情与在线咨询（大学生 → 管理员）

- 点击需求卡片，查看详细信息
- 点击"在线咨询"按钮，进入聊天窗口
- 发送消息给管理员表达接单意愿

#### 阶段五：管理员处理咨询

- 管理员在消息中心实时收到咨询
- 在线回复，了解大学生情况
- 协调对接，互相交换联系方式

#### 阶段六：后续对接

- 大学生和家长自行联系
- 安排试讲、确定时间

## 四、智能文字解析详解

### 4.1 解析逻辑流程

```
输入：粘贴的需求文字
    ↓
步骤1：按分隔符拆分多条需求
（检测：空行、"编号："、"————"等分隔符）
    ↓
步骤2：对每条需求逐个解析
    ↓
步骤3：提取字段（正则表达式 + 关键词匹配）
  - 原始编号
  - 年级
  - 性别
  - 科目
  - 薪资
  - 时间
  - 地址（区域 + 具体地点）
  - 要求
    ↓
步骤4：生成结构化数据
    ↓
步骤5：返回预览结果（JSON）
    ↓
输出：可编辑的预览表格
```

### 4.2 核心正则表达式

#### 提取原始编号

```javascript
// 匹配：编号：KKJ231、编号:KKJ229-2、KKJ231#暑假预约
/编号[：:]\s*([A-Z0-9\-]+)(?:[＃#](.*))?/i

// 示例：
"编号：KKJ231＃暑假预约" → 提取：KKJ231，标签：暑假预约
"编号:KKJ229-2" → 提取：KKJ229-2
```

#### 提取年级

```javascript
// 匹配：准六年级、小学三年级、初二、高三
/准?([一二三四五六七八九十小初高]+)年级/

// 或更智能的匹配：
/(准|小学|初中|高中)?([一二三四五六七八九十\d]+)年级/

// 示例：
"准六年级男生" → 提取：准六年级 → 转换：小学（六年级）
"初二男生" → 提取：初二 → 转换：初中（二年级）
```

#### 提取科目

```javascript
// 关键词匹配（支持组合）
const subjects = ['语文', '数学', '英语', '物理', '化学', '生物', '历史', '地理', '政治', '语数外', '全科'];

// 特殊处理：
"语数外" → 拆分为：['语文', '数学', '英语']
"全科" → 标记为全科辅导
```

#### 提取薪资

```javascript
// 匹配：140/2小时、210元/3小时、150元每小时
/(\d+)[元]?\/(\d+)小时/g

// 示例：
"140/2小时，210/3小时" → 提取：[{amount: 140, hours: 2}, {amount: 210, hours: 3}]
"150元/小时" → 提取：{amount: 150, hours: 1}
```

#### 提取地址

```javascript
// 步骤1：提取区域
const districts = ['上城区', '西湖区', '拱墅区', '余杭区', '滨江区', '萧山区', '临平区', '钱塘区'];
// 匹配任意区域关键词

// 步骤2：提取具体地点
/(?:地址|地点)[：:]\s*(.*)/

// 示例：
"地址：上城区金色黎明" → 区域：上城区，地点：金色黎明
"余杭区金家渡地铁站附近" → 区域：余杭区，地点：金家渡地铁站附近
```

#### 提取要求

```javascript
// 匹配：要求：女生，认真负责
/要求[：:]\s*(.*)/

// 进一步提取性别要求
/(女生|男生|男女不限|不限)/

// 示例：
"要求：女生，认真负责" → 性别要求：女生，其他要求：认真负责
```

// parser/index.js
const parse = (text) => {
  // 1. 拆分多条需求
  const blocks = splitter.split(text);
  
  // 2. 逐条解析
  const results = blocks.map(block => {
    // 提取字段
    const extracted = extractor.extract(block);
    
    // 映射关键词
    const mapped = mapper.map(extracted);
    
    // 验证数据
    const validated = validator.validate(mapped);
    
    return validated;
  });
  
  return results;
};

#### 6.3.2 智能文字解析实现

**核心代码示例**：

```javascript
// parser/extractor.js
const extractFields = (text) => {
  const result = {};
  
  // 提取原始编号
  const noMatch = text.match(/编号[：:]\s*([A-Z0-9\-]+)(?:[＃#](.*))?/i);
  if (noMatch) {
    result.original_no = noMatch[1];
    result.tag = noMatch[2] || '';
  }
  
  // 提取年级
  const gradeMatch = text.match(/(准|小学|初中|高中)?([一二三四五六七八九十\d]+)年级/);
  if (gradeMatch) {
    result.grade_detail = gradeMatch[0];
    // 进一步映射到小学/初中/高中
    result.grade = mapGrade(gradeMatch[0]);
  }
  
  // 提取科目
  const subjects = [];
  const subjectKeywords = ['语文', '数学', '英语', '物理', '化学', '生物', '历史', '地理', '政治', '语数外'];
  subjectKeywords.forEach(keyword => {
    if (text.includes(keyword)) {
      if (keyword === '语数外') {
        subjects.push('语文', '数学', '英语');
      } else {
        subjects.push(keyword);
      }
    }
  });
  result.subjects = [...new Set(subjects)]; // 去重
  
  // 提取薪资
  const salaryMatches = text.matchAll(/(\d+)[元]?\/(\d+)小时/g);
  result.salary_options = [];
  for (const match of salaryMatches) {
    result.salary_options.push({
      amount: parseInt(match[1]),
      hours: parseInt(match[2])
    });
  }
  
  // 提取地址
  const districts = ['上城区', '西湖区', '拱墅区', '余杭区', '滨江区', '萧山区', '临平区', '钱塘区'];
  for (const district of districts) {
    if (text.includes(district)) {
      result.district = district;
      break;
    }
  }
  
  const addressMatch = text.match(/(?:地址|地点)[：:]\s*(.*)/);
  if (addressMatch) {
    result.address = addressMatch[1].trim();
  }
  
  // 提取要求
  const reqMatch = text.match(/要求[：:]\s*(.*)/);
  if (reqMatch) {
    result.requirements = reqMatch[1].trim();
    // 提取性别要求
    if (reqMatch[1].includes('女生')) result.gender_req = '女生';
    else if (reqMatch[1].includes('男生')) result.gender_req = '男生';
    else result.gender_req = '不限';
  }
  
  return result;
};
```
